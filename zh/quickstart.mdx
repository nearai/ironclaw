前往 https://agent.near.ai/ 并使用您偏好的方式登录，然后开始创建 IronClaw 智能体。

## 创建智能体

<Steps>

<Step title="创建智能体">

创建和部署智能体非常简单，只需填写必要的计费信息、添加 SSH 密钥，然后点击"Activate Agent"按钮。智能体将自动创建并部署在私有实例中，您可以通过 SSH 连接到它。

<Accordion title="SSH 密钥">

使用 IronClaw 需要提供 SSH 密钥。如果您还没有，可以在终端中使用以下命令生成：

```bash
ssh-keygen -t rsa -b 4096 -C "you@example.com"
cat ~/.ssh/id_rsa.pub
```

</Accordion>

</Step>
<Step title="连接到您的私有实例">

使用 [智能体仪表盘](https://agent.near.ai/) 中提供的地址，通过 `SSH` 连接到智能体的私有实例：

```bash
> ssh -p <PORT> liquid-horse@agent2.near.ai

#Linux 522eab21e8e4 6.9.0-dstack #1 SMP PREEMPT_DYNAMIC Thu May 30 14:26:20 UTC 2024 x86_64

#Debian GNU/Linux comes with ABSOLUTELY NO WARRANTY, to the extent permitted by applicable law.
agent@1v4d3v23e8f1:~$
```

<Accordion title="连接遇到问题？">
连接前请确保已将 SSH 密钥添加到设备的 SSH 代理中：

```bash
ssh-add ~/.ssh/id_rsa
```

</Accordion>

</Step>

<Step title="启动智能体">

成功连接到智能体的私有实例后，可以首次启动智能体：

```bash
ironclaw
```

<Accordion title="⚠️ 错误：另一个 IronClaw 实例已在运行">

如果出现错误 `Error: Another IronClaw instance is already running (PID 38). If this is incorrect, remove the stale PID file: /home/agent/.ironclaw/ironclaw.pid`，只需运行以下命令删除过期的 PID 文件，然后重新启动智能体：

```
# 删除过期的 PID 文件
rm /home/agent/.ironclaw/ironclaw.pid

# 然后重新启动智能体
ironclaw
```

</Accordion>

由于这是首次启动智能体，它会要求您配置推理提供商和要使用的 LLM。

![setup](/images/quickstart/setup-wizard.png)

<Note>
我们推荐使用 [NEAR AI](https://cloud.near.ai/) 作为推理提供商，以获得最高的隐私和安全性，并使用 `Qwen3-30B` 模型，性价比最优。
</Note>

<Accordion title="⚠️ 错误：Webhook 服务器启动失败">

如果遇到错误 `Error: Channel webhook_server failed to start: Failed to bind to 0.0.0.0:8080: Address already in use (os error 98)`，请尝试设置不同的 HTTP 端口：

```
# 将默认 HTTP 端口更改为 8081
export HTTP_PORT=8081

# 然后重新启动智能体
ironclaw
```

</Accordion>

</Step>

<Step title="与智能体对话">

智能体启动运行后，您可以通过终端开始与它交互，只需输入您的消息，智能体就会回复。

![hello-ai](/images/quickstart/hello-ai.png)


</Step>

<Step title="更新 IronClaw">

最后，请定期更新 IronClaw 以获取最新功能和改进。您可以在终端中运行以下命令来更新：

```bash
ironclaw-update
```

</Step>

</Steps>
