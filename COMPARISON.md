# IronClaw vs OpenClaw 对比

本文档对比 IronClaw 和 OpenClaw 两个开源 AI 助手框架，帮助开发者选择适合自己需求的框架。

## 概述

| 特性 | IronClaw | OpenClaw |
|------|----------|----------|
| **编程语言** | Rust | TypeScript/Node.js |
| **核心目标** | 隐私安全 | 灵活性与扩展性 |
| **数据存储** | 本地 PostgreSQL | 可配置 |
| **安全模型** | WASM 沙箱 + 多层防护 | Docker 沙箱隔离 |

## 详细对比

### 1. 架构设计

**IronClaw**
- Rust 编写的核心，提供高性能和内存安全
- WASM 沙箱运行不受信任的工具
- PostgreSQL + pgvector 实现混合搜索

**OpenClaw**
- TypeScript/Node.js，生态丰富
- 插件系统，支持多种 channel
- 更灵活的部署选项

### 2. 安全特性

**IronClaw**
- Defense in depth: 多层安全防护
- Prompt injection 防御
- 凭证保护：密钥从不在工具间暴露
- 端点白名单

**OpenClaw**
- 插件权限管理
- 可配置的 channel 安全
- MCP 协议支持

### 3. 扩展性

**IronClaw**
- 动态工具构建 (WASM)
- MCP 协议集成
- 多 provider 支持 (OpenAI, Anthropic, Ollama 等)

**OpenClaw**
- 丰富的 skills 系统
- 插件市场
- 多 channel 支持 (Telegram, Slack, Discord 等)

### 4. 部署

**IronClaw**
- Docker 支持
- 需要 PostgreSQL 15+ 及 pgvector 扩展
- Rust 1.85+

**OpenClaw**
- 更轻量级部署
- 更多部署选项

## 使用场景推荐

### 选择 IronClaw 当:
- 隐私是首要考虑
- 需要高性能
- 对 Rust 生态熟悉
- 需要强安全防护

### 选择 OpenClaw 当:
- 需要快速原型
- 偏好 JavaScript/TypeScript
- 需要丰富的 channel 集成
- 社区支持更重要

## 结论

两个项目都代表了 AI 助手框架的不同设计哲学：

- **IronClaw** = 隐私优先 + 安全优先
- **OpenClaw** = 灵活优先 + 生态优先

根据具体需求选择合适的框架。
