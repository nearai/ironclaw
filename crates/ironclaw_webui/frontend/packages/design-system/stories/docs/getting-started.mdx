# IronClaw Design System

React components built on Radix with shadcn-style patterns, the `--v2-*`
design tokens, and a small motion vocabulary. The agent workspace consumes
this package today; anything else we build (marketing, sub-UIs, other
repos) can import the same package and look like IronClaw.

The point of all this: when the design decisions are already made and
enforced by tokens, small UI changes stop needing a design review. An
engineer — or an agent — can ship them safely. This Storybook is the
living spec.

## How it's put together

- **Radix underneath.** Dialog, Select, DropdownMenu, Tabs, Tooltip,
  Popover, Checkbox, Switch, RadioGroup, Slider, Avatar, Separator,
  ScrollArea. Keyboard navigation, focus management, and ARIA come from
  the primitive, not from code we wrote.
- **shadcn on top.** CVA variants, `cn()` (clsx + tailwind-merge), and
  Tailwind classes bound to CSS custom properties.
- **Tokens, not values.** Colors, radii, control heights, spacing, type
  sizes, shadows, durations, z-layers — all `--v2-*` custom properties.
  Components never restate a raw value.

## Installing it

Inside the ironclaw repo the app already gets it through pnpm workspaces
(`"@ironclaw/design-system": "workspace:*"`). From another repo, link a
local checkout or add it as a file/git dependency:

```bash
# link a local checkout while developing
pnpm link /path/to/ironclaw/crates/ironclaw_webui/frontend/packages/design-system

# or pin it as a file dependency
# "@ironclaw/design-system": "file:../ironclaw/crates/ironclaw_webui/frontend/packages/design-system"
```

The package ships TypeScript source (`exports` point at `.ts`), which any
Vite-style bundler handles out of the box. If yours can't, `pnpm build`
emits a compiled ESM bundle with declarations in `dist/`.

## Wiring it up

```css
/* your app.css — Tailwind v4 host */
@import "tailwindcss";
@import "@ironclaw/design-system/tokens.css";
@source "../node_modules/@ironclaw/design-system/src";
```

Set `data-theme="light" | "dark"` on `<html>`, or use the exported
`useInterfaceTheme` hook. If your app is localized, pass your translator
through `DesignSystemI18nProvider` — otherwise the couple of built-in
strings (modal close, confirm cancel) fall back to English.

## Using it

```tsx
import { Button, Card, Modal, SelectMenu, Tabs } from "@ironclaw/design-system";
import { COLOR_TOKENS, STATUS_CANON } from "@ironclaw/design-system/tokens";
import { MOTION_DURATION } from "@ironclaw/design-system/motion";
```

Every component page here shows the variants, a props table, and the
story source (`Show code` under each canvas) — copy-paste from those. The
`/playground` route in the running WebUI renders the same components
inside the real app shell if you want to check something against actual
chrome.
