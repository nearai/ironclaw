IronClaw 可连接受支持的 [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) 服务器，并把其工具暴露给代理。MCP 是开放标准，生态中已有大量数据库、API、云服务连接器。

<Info>
下方的 `ironclaw mcp ...` 命令配置 v1 `ironclaw` 二进制；它支持 HTTP、
stdio 和 Unix 套接字传输。Reborn 改用扩展清单：`ironclaw-reborn` 仅接受
host-bundled 包的远程 HTTPS MCP，以及 installed-local 包指向精确 IPv4
回环地址（`127.0.0.0/8`）的明文 HTTP MCP。它拒绝 `localhost`、IPv6、
局域网/非回环私网地址、远程明文、registry-installed 和 stdio MCP 端点；
省略回环 HTTP 端口时固定为 80。
</Info>

---

## 添加服务器

可直接要求代理使用 MCP，或通过 CLI 添加：

```bash
ironclaw mcp add <server_name> <server_url>
```

---

## 认证

如果服务器需要认证：

```bash
ironclaw mcp auth <server_name>
```

---

## 查看已连接服务器

```bash
ironclaw mcp list
```

连接成功后，MCP 工具会出现在代理工具列表中。

---

## 移除服务器

```bash
ironclaw mcp remove <server_name>
```

---

## WASM 与 MCP 如何选择

| 维度 | WASM | MCP |
|------|------|-----|
| 隔离性 | 强（wasmtime 沙箱、fuel、内存限制） | 位于 wasmtime 沙箱之外的外部服务或进程 |
| 凭据处理 | 代理层注入，模块看不到原始密钥 | 由 MCP 服务自行处理 |
| 网络控制 | `capabilities.json` 白名单 | Reborn 将主机出站锁定到清单端点；MCP 服务控制自身出站访问 |
| 生态 | 自建为主 | 现成生态丰富 |
| 语言 | 任意 `wasm32-wasi` 目标 | 任意语言 |
| 启动成本 | 首次需编译缓存 | MCP 服务需预先运行 |
| 适合场景 | 强隔离的定制集成 | 复用现有 MCP 服务 |

<Note>
涉及敏感凭据或不可信外部数据时，优先使用 WASM 工具，可获得更强隔离保障。
</Note>
