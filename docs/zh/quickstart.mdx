## 设置您的智能体

<Steps>

<Step title="获取 IronClaw">

<Tabs>
  <Tab title="Agent Hub">
  前往 https://agent.near.ai/ 并使用您偏好的方式登录，然后在私有实例中创建一个 IronClaw 智能体。

  私有实例准备就绪后，您可以通过[智能体仪表盘](https://agent.near.ai/)提供的地址使用 `SSH` 连接：

  ```bash
  ssh -p <PORT> liquid-horse@agent2.near.ai
  ```

  <Accordion title="SSH 密钥">

  如果您还没有 SSH 密钥，可以在终端中生成：

  ```bash
  ssh-keygen -t rsa -b 4096 -C "you@example.com"
  cat ~/.ssh/id_rsa.pub
  ```

  </Accordion>

  <Accordion title="连接遇到问题？">

  连接前请确保已将 SSH 密钥添加到设备的 SSH 代理中：

  ```bash
  ssh-add ~/.ssh/id_rsa
  ```

  </Accordion>
  </Tab>

  <Tab title="本地">
    适合在自己的机器上个人使用。默认 `local-dev` profile 将 Reborn 状态保存在 `~/.ironclaw/reborn`，不需要 retained v1 的数据库配置。托管和生产存储请参阅 [canonical CLI 参考](/reborn-binary#run)。

    ```bash
    # 安装 IronClaw
    curl --proto '=https' --tlsv1.2 -LsSf https://github.com/nearai/ironclaw/releases/latest/download/ironclaw-installer.sh | sh
    ```
  </Tab>
</Tabs>

</Step>

<Step title="初始化智能体">

初始化 canonical Reborn runtime。命令会在 `~/.ironclaw/reborn` 下创建隔离的配置、provider 和 WebUI token 文件，不会修改 retained v1 状态：

```bash
ironclaw onboard
ironclaw config path
```

选择 provider 和模型，然后检查生效的 route：

```bash
ironclaw models set-provider openai --model gpt-5-mini
ironclaw models status
```

启动运行前，请设置所选 provider 要求的 API key 环境变量。

</Step>

<Step title="与智能体对话">

启动交互式 REPL，或发送一条消息后退出：

```bash
ironclaw repl
ironclaw run --message "你好"
```

</Step>

<Step title="更新 IronClaw">

请定期更新 IronClaw，以获得最新功能和修复：

```bash
ironclaw-update
```

</Step>

</Steps>
