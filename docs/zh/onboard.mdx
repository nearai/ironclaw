`ironclaw onboard` 用于初始化 canonical Reborn runtime。它是非交互式 bootstrap 命令，不是 retained v1 的一站式设置向导。

<Note>
如果您还没有安装 IronClaw，请先阅读[快速开始指南](/zh/quickstart)。
</Note>

## 命令执行的操作

```bash
ironclaw onboard
```

该命令会：

1. 读取 `IRONCLAW_REBORN_HOME`，未设置时使用 `~/.ironclaw/reborn`；
2. 创建 Reborn home 目录；
3. 创建缺失的 `config.toml` 和 `providers.json`；
4. 默认保留运营人员已经编辑的文件，只有传入 `--force` 才覆盖；
5. 生成或保留受文件权限保护的 `webui-token` bearer token；
6. 写入 `.onboard-completed.json`，记录已完成和待完成的设置步骤。

它不会读取或修改 retained v1 状态，也不会配置 v1 数据库，或运行 v1 的频道、隧道、扩展、沙箱和 heartbeat 向导。

## 完成 provider 设置

选择 provider 和模型，然后检查生效的 route：

```bash
ironclaw models list
ironclaw models set-provider openai --model gpt-5-mini
ironclaw models status
```

启动 runtime 前，请设置 `models list --verbose` 显示的 API key 环境变量。LLM/API credential 保留在环境变量中，不会写入 `config.toml`；生成的 WebUI bearer token 存放在受文件权限保护的 `webui-token` 中。

## 检查生成的配置

```bash
ironclaw config path
ironclaw config list
ironclaw config get boot.profile
```

只有在明确要替换生成的配置 stub 时，才使用 `ironclaw config init --force`。

## 启动 runtime

```bash
ironclaw repl
ironclaw run --message "你好"
```

Profile、生产存储、WebUI serve 模式和所有命令选项请参阅 [canonical CLI 参考](/reborn-binary)。详细 onboarding contract 和当前 non-goals 请参阅 [Reborn onboarding](/reborn/onboarding)。

<Warning>
Retained v1 向导只能通过 `ironclaw-v1 onboard` 使用。它操作 v1 状态，不属于 canonical Reborn onboarding 流程。
</Warning>
