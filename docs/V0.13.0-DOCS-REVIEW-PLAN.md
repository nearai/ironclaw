# IronClaw v0.13.0 Documentation Integrity Review Plan

## Goal

Validate that `ironclaw-docs` accurately reflects IronClaw `v0.13.0` source behavior across CLI, HTTP API, runtime configuration, and onboarding flow, then correct factual mismatches.

## Sources

Source baseline:
- `~/src/ironclaw` at tag `v0.13.0` (commit checks in local repo).

Documentation baseline:
- `~/src/ironclaw-docs` at current branch `main` (targeted for `v0.13.0-docs.*` release).

Online validation:
- GitHub repository links and release pages for version/tag context.
- Crate/package metadata pages for drift checks where authoritative sources are accessible.

## Review workflow (detailed)

1. Enumerate public contracts in `src/`:
   - CLI flags and subcommands in `src/main.rs`, `src/setup/`, `src/channels`.
   - Environment/config defaults in `src/config/*.rs`.
   - Web gateway endpoints in `src/channels/web/server.rs`.
   - Tool/extension APIs in `src/tools`, `src/extensions`, `src/skills`.
2. Compare each contract against docs:
   - `analysis/config.md`
   - `analysis/channels.md`
   - `analysis/cli.md`
   - `INSTALLATION.md`
   - `AGENT_README.md`
   - `ARCHITECTURE.md`
3. For every mismatch, classify:
   - stale example value
   - undocumented route/option
   - broken narrative claim
   - version/context mismatch
4. Apply doc-only fixes with minimal behavior changes.
5. Re-run diff sanity check:
   - ensure no edits to unrelated docs sections unless explicitly part of a correction.
6. Tag release in docs repo only after all factual corrections are applied.
