Complete reference for the `ironclaw` CLI.

## Global Options

```bash
ironclaw [OPTIONS] [COMMAND]
```

| Option | Description |
|--------|-------------|
| `-c, --config <FILE>` | Path to config file |
| `--no-onboard` | Skip auto-onboarding |
| `-v, --verbose` | Enable verbose logging |
| `-h, --help` | Print help |
| `-V, --version` | Print version |

## Commands

### run

Start the IronClaw agent.

```bash
ironclaw run [OPTIONS]
```

Starts the agent with all configured channels. This is the main command for normal operation.

**Options:**

| Option | Description |
|--------|-------------|
| `--no-tui` | Disable TUI, use HTTP only |

**Examples:**

```bash
# Start normally
ironclaw run

# Start without TUI
ironclaw run --no-tui

# With verbose logging
RUST_LOG=ironclaw=debug ironclaw run
```

### onboard

Run the interactive setup wizard.

```bash
ironclaw onboard [OPTIONS]
```

Configures database, LLM, channels, and security settings.

**Options:**

| Option | Description |
|--------|-------------|
| `--skip-auth` | Skip authentication steps |
| `--channels-only` | Configure only channels |

**Examples:**

```bash
# Full wizard
ironclaw onboard

# Skip auth (use existing)
ironclaw onboard --skip-auth

# Add new channel
ironclaw onboard --channels-only
```

### config

Manage configuration settings.

```bash
ironclaw config <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list` | List all settings |
| `get <KEY>` | Get a specific setting |
| `set <KEY> <VALUE>` | Set a setting |
| `delete <KEY>` | Delete a setting |

**Examples:**

```bash
# List all settings
ironclaw config list

# Get LLM backend
ironclaw config get llm.backend

# Set model
ironclaw config set llm.model claude-sonnet-4

# Delete setting (reset to default)
ironclaw config delete llm.model
```

### traces

Manage opt-in trace contributions.

```bash
ironclaw traces <SUBCOMMAND>
```

Trace contribution is disabled by default. `preview` converts an existing recorded trace into a redacted contribution envelope with consent, redaction, trace-card, value-card, scorecard, replayability, revocation, and credit metadata. Autonomous submission only runs after `opt-in` configures an explicit private ingestion endpoint.

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `opt-in` | Enable autonomous submission after local redaction |
| `opt-out` | Disable autonomous submission |
| `status` | Show the local contribution policy |
| `preview` | Build a redacted contribution envelope from a recorded trace |
| `enqueue` | Queue a redacted envelope for autonomous submission |
| `flush-queue` | Submit eligible queued envelopes |
| `credit` | Show local credit totals and explanations |
| `submit` | Submit one redacted envelope to an explicit endpoint |
| `list-submissions` | List local submission records |
| `revoke` | Mark a submission revoked locally and optionally call a revocation endpoint |
| `ingest-health` | Check a Trace Commons ingestion service `/health` endpoint |
| `quarantine-list` | List centrally quarantined traces for reviewer/admin triage |
| `active-learning-review-queue` | List active-learning central trace review work |
| `review-decision` | Approve or reject a central quarantined trace |
| `append-credit-event` | Append a delayed central credit ledger event |
| `analytics-summary` | Show central Trace Commons analytics |
| `maintenance-run` | Request central retention/revocation propagation and DB reconciliation maintenance |
| `benchmark-convert` | Request central benchmark conversion for approved replayable traces |
| `replay-dataset-export` | Export approved replay dataset metadata |
| `ranker-training-candidates` | Export approved ranker training candidates |
| `ranker-training-pairs` | Export approved ranker training pairs |
| `audit-events` | List central Trace Commons audit events |
| `list-traces` | List central trace metadata with reviewer filters |
| `privacy-filter-canary` | Run a local Privacy Filter sidecar canary check |

**Examples:**

```bash
ironclaw traces opt-in --endpoint https://trace-ingest.internal/v1/traces
ironclaw traces preview --recorded-trace trace.json --output contribution.json
ironclaw traces enqueue --envelope contribution.json
ironclaw traces flush-queue
ironclaw traces credit
ironclaw traces ingest-health --endpoint https://trace-ingest.internal
ironclaw traces quarantine-list --endpoint https://trace-ingest.internal --bearer-token-env TRACE_COMMONS_REVIEWER_TOKEN
ironclaw traces active-learning-review-queue --endpoint https://trace-ingest.internal --limit 25 --privacy-risk medium
ironclaw traces review-decision 018f2b7b-0c11-72fd-95c4-1f9f98feac01 --endpoint https://trace-ingest.internal --decision approve --reason "redaction verified"
ironclaw traces append-credit-event 018f2b7b-0c11-72fd-95c4-1f9f98feac01 --endpoint https://trace-ingest.internal --event-type regression-catch --credit-points-delta 2.5 --reason "Caught replay regression" --external-ref regression:trace-replay-smoke
ironclaw traces analytics-summary --endpoint https://trace-ingest.internal --json
ironclaw traces maintenance-run --endpoint https://trace-ingest.internal --purpose db-read-cutover --dry-run --reconcile-db-mirror
ironclaw traces benchmark-convert --endpoint https://trace-ingest.internal --purpose regression-suite --consent-scope benchmark-only --status accepted --privacy-risk low --limit 100
ironclaw traces replay-dataset-export --endpoint https://trace-ingest.internal --purpose regression-suite --consent-scope benchmark-only --privacy-risk low --limit 100 --output replay-dataset.json
ironclaw traces ranker-training-candidates --endpoint https://trace-ingest.internal --consent-scope ranking-training --privacy-risk low --limit 100 --output ranker-candidates.json
ironclaw traces ranker-training-pairs --endpoint https://trace-ingest.internal --consent-scope ranking-training --privacy-risk low --limit 100 --output ranker-pairs.json
ironclaw traces audit-events --endpoint https://trace-ingest.internal --limit 50
ironclaw traces list-traces --endpoint https://trace-ingest.internal --status accepted --privacy-risk low
ironclaw traces privacy-filter-canary
```

`flush-queue` submits eligible queued traces under the standing opt-in policy and emits a credit notice when the configured interval is due. `credit` reports local pending/final credit and recent explanations. Central delayed credit events, such as benchmark use or regression catches, can be returned by the private ingestion API and reflected in the local ledger.

Central trace status filters accept `accepted`, `quarantined`, `rejected`, `revoked`, `expired`, and `purged`.

Central reviewer/admin commands always require an explicit endpoint. Health accepts either a service base URL, `/health`, or `/v1/traces` URL; other central commands accept a service base URL or `/v1/traces` URL and derive the expected private API paths. Reviewer/admin commands use `--bearer-token-env` and fail cleanly on missing credentials or non-2xx responses, including older servers that do not yet implement maintenance, benchmark conversion, active-learning review queue, or ranker training export endpoints. They do not change local opt-in defaults and do not upload raw traces.

`active-learning-review-queue` calls `/v1/review/active-learning` and exits normally with a not-exposed-yet message when an older service returns 404 or 501. `maintenance-run` posts to `/v1/admin/maintenance` with an explicit `--purpose` and optional `--dry-run`; `--purge-expired-before <RFC3339>` opt-in purges expired submissions at or before that cutoff, and `--reconcile-db-mirror` prints compact metadata counts plus reader-projection parity for DB read-cutover checks. `benchmark-convert` posts to `/v1/benchmarks/convert` with `--purpose` plus consent/status/privacy filters. `replay-dataset-export` accepts the same export filters where supported and prints export, manifest, and audit identifiers when the server returns them. `ranker-training-candidates` calls `/v1/ranker/training-candidates`; `ranker-training-pairs` calls `/v1/ranker/training-pairs`. Both ranker commands accept consent/privacy/limit filters, can write the returned JSON with `--output`, and exit normally on 404 or 501 for services that have not added the endpoints. `append-credit-event` requires `--reason`; benchmark, regression, training, and ranking utility events also require `--external-ref` so delayed credit is linked to a downstream artifact or job. `privacy-filter-canary` is local only; it uses `IRONCLAW_TRACE_PRIVACY_FILTER_COMMAND` and optional `IRONCLAW_TRACE_PRIVACY_FILTER_ARGS` and refuses to pass without a configured sidecar. Sidecar timeout and IO limits can be tuned with `IRONCLAW_TRACE_PRIVACY_FILTER_TIMEOUT_MS`, `IRONCLAW_TRACE_PRIVACY_FILTER_MAX_INPUT_BYTES`, `IRONCLAW_TRACE_PRIVACY_FILTER_MAX_STDOUT_BYTES`, and `IRONCLAW_TRACE_PRIVACY_FILTER_MAX_STDERR_BYTES`.

### registry

Manage the tool registry.

```bash
ironclaw registry <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list` | List available tools |
| `search <QUERY>` | Search for tools |
| `install <NAME>` | Install from registry |
| `update` | Update registry index |

**Examples:**

```bash
# List available tools
ironclaw registry list

# Search
ironclaw registry search calendar

# Install
ironclaw registry install google-calendar
```

### mcp

Manage MCP (Model Context Protocol) servers.

```bash
ironclaw mcp <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list` | List MCP servers |
| `add <URL>` | Add an MCP server |
| `remove <NAME>` | Remove an MCP server |

**Examples:**

```bash
ironclaw mcp list
ironclaw mcp add http://localhost:3000/sse
```

### memory

Manage workspace memory.

```bash
ironclaw memory <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list` | List documents |
| `read <PATH>` | Read a document |
| `write <PATH>` | Write a document |
| `search <QUERY>` | Search memory |
| `tree` | Show memory tree |

**Examples:**

```bash
# List documents
ironclaw memory list

# Write a document
echo "My note" | ironclaw memory write notes/idea.md

# Search
ironclaw memory search "project idea"
```

### pairing

Manage channel pairing.

```bash
ironclaw pairing <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list <CHANNEL>` | List pending requests |
| `approve <CHANNEL> <CODE>` | Approve a pairing request |

**Examples:**

```bash
# List Telegram pending
ironclaw pairing list telegram

# Approve
ironclaw pairing approve telegram ABC12345

# List as JSON
ironclaw pairing list telegram --json
```

### doctor

Run diagnostics.

```bash
ironclaw doctor [OPTIONS]
```

Checks:
- Database connectivity
- LLM provider access
- Docker availability
- Tunnel detection

**Options:**

| Option | Description |
|--------|-------------|
| `--json` | Output as JSON |
| `--fix` | Attempt to fix issues |

**Example:**

```bash
ironclaw doctor

```

Show current status.

```bash
ironclaw status [OPTIONS]
```

Shows:
- Version
- Configuration
- Database status
- LLM backend
- Channels

**Example:**

```bash
ironclaw status
```

### completion

Generate shell completion scripts.

```bash
ironclaw completion <SHELL>
```

**Shells:** `bash`, `zsh`, `fish`, `powershell`

**Example:**

```bash
# Bash
ironclaw completion bash > /etc/bash_completion.d/ironclaw

# Zsh
ironclaw completion zsh > /usr/local/share/zsh/site-functions/_ironclaw
```

## Environment Variables

CLI behavior can be modified with environment variables:

```bash
# Config directory
export IRONCLAW_BASE_DIR=/custom/path

# Skip onboarding
export ONBOARD_COMPLETED=true

# Logging
export RUST_LOG=ironclaw=debug
```

## Exit Codes

| Code | Meaning |
|------|---------|
| `0` | Success |
| `1` | General error |
| `2` | Invalid arguments |
| `3` | Configuration error |
| `4` | Database error |
| `5` | Network error |
