Complete reference for the `ironclaw` CLI.

## Global Options

```bash
ironclaw [OPTIONS] [COMMAND]
```

| Option | Description |
|--------|-------------|
| `-c, --config <FILE>` | Path to config file |
| `--no-onboard` | Skip auto-onboarding |
| `-v, --verbose` | Enable verbose logging |
| `-h, --help` | Print help |
| `-V, --version` | Print version |

## Commands

### run

Start the IronClaw agent.

```bash
ironclaw run [OPTIONS]
```

Starts the agent with all configured channels. This is the main command for normal operation.

**Options:**

| Option | Description |
|--------|-------------|
| `--no-tui` | Disable TUI, use HTTP only |

**Examples:**

```bash
# Start normally
ironclaw run

# Start without TUI
ironclaw run --no-tui

# With verbose logging
RUST_LOG=ironclaw=debug ironclaw run
```

### onboard

Run the interactive setup wizard.

```bash
ironclaw onboard [OPTIONS]
```

Configures database, LLM, channels, and security settings.

**Options:**

| Option | Description |
|--------|-------------|
| `--skip-auth` | Skip authentication steps |
| `--channels-only` | Configure only channels |

**Examples:**

```bash
# Full wizard
ironclaw onboard

# Skip auth (use existing)
ironclaw onboard --skip-auth

# Add new channel
ironclaw onboard --channels-only
```

### config

Manage configuration settings.

```bash
ironclaw config <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list` | List all settings |
| `get <KEY>` | Get a specific setting |
| `set <KEY> <VALUE>` | Set a setting |
| `delete <KEY>` | Delete a setting |

**Examples:**

```bash
# List all settings
ironclaw config list

# Get LLM backend
ironclaw config get llm.backend

# Set model
ironclaw config set llm.model claude-sonnet-4

# Delete setting (reset to default)
ironclaw config delete llm.model
```

### tool

```bash
ironclaw tool <SUBCOMMAND>
```

| Subcommand | Description |

**Examples:**


### traces

Manage opt-in trace contributions.

```bash
ironclaw traces <SUBCOMMAND>
```

Trace contribution is disabled by default. `preview` converts an existing recorded trace into a redacted contribution envelope with consent, redaction, trace-card, value-card, scorecard, replayability, revocation, and credit metadata. Autonomous submission only runs after `opt-in` configures an explicit private ingestion endpoint.

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `opt-in` | Enable autonomous submission after local redaction |
| `opt-out` | Disable autonomous submission |
| `status` | Show the local contribution policy |
| `preview` | Build a redacted contribution envelope from a recorded trace |
| `enqueue` | Queue a redacted envelope for autonomous submission |
| `flush-queue` | Submit eligible queued envelopes |
| `queue-status` | Show local autonomous queue diagnostics |
| `credit` | Show local credit totals and explanations |
| `submit` | Submit one redacted envelope to an explicit endpoint |
| `list-submissions` | List local submission records |
| `revoke` | Mark a submission revoked locally and optionally call a revocation endpoint |
| `ingest-health` | Check a Trace Commons ingestion service `/health` endpoint |
| `quarantine-list` | List centrally quarantined traces for reviewer/admin triage |
| `active-learning-review-queue` | List active-learning central trace review work |
| `review-decision` | Approve or reject a central quarantined trace |
| `review-lease-claim` | Claim a DB-backed central trace review lease |
| `review-lease-release` | Release a DB-backed central trace review lease |
| `append-credit-event` | Append a delayed central credit ledger event |
| `worker-utility-credit` | Append idempotent delayed utility credit through the worker route |
| `process-evaluation-submit` | Submit process-evaluation metadata and optional idempotent training utility credit |
| `analytics-summary` | Show central Trace Commons analytics |
| `maintenance-run` | Request central retention/revocation propagation and DB reconciliation maintenance |
| `worker-retention-maintenance` | Request scoped retention maintenance through the worker route |
| `retention-jobs-list` | List DB-backed retention maintenance jobs |
| `retention-job-items` | List DB-backed per-submission retention job actions |
| `worker-vector-index` | Request scoped vector indexing through the worker route |
| `benchmark-convert` | Request central benchmark conversion for approved replayable traces |
| `worker-benchmark-convert` | Request benchmark conversion through the worker route |
| `benchmark-lifecycle-update` | Update benchmark registry/evaluator lifecycle metadata |
| `replay-dataset-export` | Export approved replay dataset metadata |
| `replay-export-manifests` | List central replay export manifest metadata |
| `config-status` | Show safe non-secret Trace Commons config cutover/status booleans |
| `tenant-policy-get` | Read the DB-backed tenant contribution policy |
| `tenant-policy-set` | Write the DB-backed tenant contribution policy |
| `ranker-training-candidates` | Export approved ranker training candidates |
| `ranker-training-pairs` | Export approved ranker training pairs |
| `audit-events` | List central Trace Commons audit events |
| `list-traces` | List central trace metadata with reviewer filters |
| `privacy-filter-canary` | Run a local Privacy Filter sidecar canary check |

`flush-queue` submits eligible queued traces under the standing opt-in policy, keeps retryable submission failures queued with typed redacted backoff sidecars, and emits a credit notice when the configured interval is due. `queue-status` reads local scoped state only and reports policy readiness, endpoint and bearer-token environment status, capture toggles, queued/held counts, retry/manual-review/policy hold counts, next retry time, durable flush/status-sync telemetry, retryable submission failure counters, sanitized held-reason counts, and the local credit summary. `credit` reports local pending/final credit and recent explanations; `credit --notice` prints and marks a due periodic notice, `credit --notice --ack` suppresses the current local credit fingerprint until credit changes, and `credit --notice --snooze-hours N` suppresses it until the snooze deadline. Central delayed credit events, such as benchmark use or regression catches, can be returned by the private ingestion API and reflected in the local ledger and periodic notice summaries, including delayed ledger deltas and credit-event counts.

Central trace status filters accept `accepted`, `quarantined`, `rejected`, `revoked`, `expired`, and `purged`.

Central reviewer/admin commands always require an explicit endpoint. Health accepts either a service base URL, `/health`, or `/v1/traces` URL; other central commands accept a service base URL or `/v1/traces` URL and derive the expected private API paths. Reviewer/admin commands use `--bearer-token-env` and fail cleanly on missing credentials or non-2xx responses, including older servers that do not yet implement maintenance, benchmark conversion, active-learning review queue, or ranker training export endpoints. They do not change local opt-in defaults and do not upload raw traces. Production upload-claim bearer tokens should be EdDSA/Ed25519 signed claims; static tokens and HS256 signed claims are internal bridge paths for controlled pilots. The ingestion service can load default/keyed EdDSA public keys and JSON/file keysets with optional `not_before`/`not_after` activation windows; managed issuer distribution/governance is still planned production work.

`active-learning-review-queue` calls `/v1/review/active-learning` and exits normally with a not-exposed-yet message when an older service returns 404 or 501; `quarantine-list` and `active-learning-review-queue` accept `--lease-filter all|mine|available|active|expired`, and non-JSON review queue and trace-list output includes lease assignment/expiry fields when the service returns them. `review-lease-claim` posts to `/v1/review/{submission_id}/lease` with optional `--lease-ttl-seconds` and `--review-due-at <RFC3339>` fields; `review-lease-release` deletes the same lease route. `maintenance-run` posts to `/v1/admin/maintenance` with an explicit `--purpose` and optional `--dry-run`; `--backfill-db-mirror`, `--index-vectors`, and `--reconcile-db-mirror` enable DB bridge work and compact reader-projection parity output, including retention job/item ledger counts and current-run retention ledger gaps, `--purge-expired-before <RFC3339>` opt-in purges expired submissions at or before that cutoff, `--max-export-age-hours <N>` bounds export-cache pruning, and `--skip-export-cache-prune` disables the default export-cache prune request. Maintenance summaries include `benchmark_artifacts_invalidated` when revoked or expired sources lifecycle-revoke published benchmark artifacts. `worker-retention-maintenance` posts only the scoped retention fields to `/v1/workers/retention-maintenance`; `retention-jobs-list` and `retention-job-items` call admin DB-backed retention ledger read endpoints, support status/action filters, and exit normally on 404 or 501 for older services. `worker-vector-index` posts only vector purpose/dry-run fields to `/v1/workers/vector-index`. `benchmark-convert` posts to `/v1/benchmarks/convert` with `--purpose` plus consent/status/privacy filters, while `worker-benchmark-convert` uses the same request shape against `/v1/workers/benchmark-convert`. `worker-utility-credit` posts `event_type`, `credit_points_delta`, `reason`, `external_ref`, and one or more `submission_ids` to `/v1/workers/utility-credit`, prints appended/skipped counts for non-JSON responses, and only accepts regression/training/ranking utility event types. `process-evaluation-submit` posts bounded rubric metadata to `/v1/workers/process-evaluation`; when `--utility-credit-points-delta` is present, `--utility-external-ref` is required and the service can append idempotent `training_utility` delayed credit for the evaluated accepted submission, with appended/skipped counts printed in non-JSON output when returned. `analytics-summary` prints content-free process-evaluation aggregates when the service returns them, including evaluated trace count, labels, rubric ratings, score bands, and any configured small-cell suppression threshold/count. `benchmark-lifecycle-update` posts registry/evaluator patches to `/v1/benchmarks/{conversion_id}/lifecycle` and supports registry status/ref, evaluator status/ref, score, pass/fail counts, timestamps, and an operator reason. `replay-dataset-export` accepts the same export filters where supported and prints export, manifest, and audit identifiers when the server returns them. `replay-export-manifests` calls `/v1/datasets/replay/manifests` and prints compact replay manifest metadata for reviewer/admin audit. `config-status` calls `/v1/admin/config-status` using an admin token and prints JSON containing safe non-secret Trace Commons cutover/status booleans plus signed-token total/EdDSA key counts, configured legal-hold retention policy IDs, the effective max export item cap, analytics min-cell threshold, and configured autonomous submission quotas. It does not expose key material, key ids, token values, issuer/audience values, or tenant policies. `tenant-policy-get` and `tenant-policy-set` call `/v1/admin/tenant-policy` using an admin token; `tenant-policy-set` sends server enum values such as `debugging_evaluation` and `ranking_model_training` while accepting CLI-friendly kebab-case arguments. `ranker-training-candidates` calls `/v1/ranker/training-candidates`; `ranker-training-pairs` calls `/v1/ranker/training-pairs`. Both ranker commands accept consent/status/privacy/limit filters, can write the returned JSON with `--output`, and exit normally on 404 or 501 for services that have not added the endpoints. `append-credit-event` requires `--reason`; benchmark, regression, training, and ranking utility events also require `--external-ref` so delayed credit is linked to a downstream artifact or job. `privacy-filter-canary` is local only; it uses `IRONCLAW_TRACE_PRIVACY_FILTER_COMMAND` and optional `IRONCLAW_TRACE_PRIVACY_FILTER_ARGS` and refuses to pass without a configured sidecar. Sidecar timeout and IO limits can be tuned with `IRONCLAW_TRACE_PRIVACY_FILTER_TIMEOUT_MS`, `IRONCLAW_TRACE_PRIVACY_FILTER_MAX_INPUT_BYTES`, `IRONCLAW_TRACE_PRIVACY_FILTER_MAX_STDOUT_BYTES`, and `IRONCLAW_TRACE_PRIVACY_FILTER_MAX_STDERR_BYTES`.

### registry

```bash
ironclaw registry <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list` | List available tools |

### completion

Generate shell completion scripts.

```bash
ironclaw completion <SHELL>
```

**Shells:** `bash`, `zsh`, `fish`, `powershell`

**Example:**

```bash
# Bash
ironclaw completion bash > /etc/bash_completion.d/ironclaw

# Zsh
ironclaw completion zsh > /usr/local/share/zsh/site-functions/_ironclaw
```

## Environment Variables

CLI behavior can be modified with environment variables:

```bash
# Config directory
export IRONCLAW_BASE_DIR=/custom/path

# Skip onboarding
export ONBOARD_COMPLETED=true

# Logging
export RUST_LOG=ironclaw=debug
```

| `1` | General error |
| `2` | Invalid arguments |
| `3` | Configuration error |
| `4` | Database error |
| `5` | Network error |
