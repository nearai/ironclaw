Complete reference for the `ironclaw` CLI.

## Global Options

```bash
ironclaw [OPTIONS] [COMMAND]
```

| Option | Description |
|--------|-------------|
| `-c, --config <FILE>` | Path to config file |
| `--no-onboard` | Skip auto-onboarding |
| `-v, --verbose` | Enable verbose logging |
| `-h, --help` | Print help |
| `-V, --version` | Print version |

## Commands

### run

Start the IronClaw agent.

```bash
ironclaw run [OPTIONS]
```

Starts the agent with all configured channels. This is the main command for normal operation.

**Options:**

| Option | Description |
|--------|-------------|
| `--no-tui` | Disable TUI, use HTTP only |

**Examples:**

```bash
# Start normally
ironclaw run

# Start without TUI
ironclaw run --no-tui

# With verbose logging
RUST_LOG=ironclaw=debug ironclaw run
```

### onboard

Run the interactive setup wizard.

```bash
ironclaw onboard [OPTIONS]
```

Configures database, LLM, channels, and security settings.

**Options:**

| Option | Description |
|--------|-------------|
| `--skip-auth` | Skip authentication steps |
| `--channels-only` | Configure only channels |

**Examples:**

```bash
# Full wizard
ironclaw onboard

# Skip auth (use existing)
ironclaw onboard --skip-auth

# Add new channel
ironclaw onboard --channels-only
```

### config

Manage configuration settings.

```bash
ironclaw config <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list` | List all settings |
| `get <KEY>` | Get a specific setting |
| `set <KEY> <VALUE>` | Set a setting |
| `delete <KEY>` | Delete a setting |

**Examples:**

```bash
# List all settings
ironclaw config list

# Get LLM backend
ironclaw config get llm.backend

# Set model
ironclaw config set llm.model claude-sonnet-4

# Delete setting (reset to default)
ironclaw config delete llm.model
```

### traces

Manage opt-in trace contributions.

```bash
ironclaw traces <SUBCOMMAND>
```

Trace contribution is disabled by default. `preview` converts an existing recorded trace into a redacted contribution envelope with consent, redaction, trace-card, value-card, scorecard, replayability, revocation, and credit metadata.

Autonomous submission only runs after `opt-in` configures an explicit private ingestion endpoint. `opt-in` writes both the global CLI policy and the runtime/web policy for this instance's configured `owner_id`; operators can target a hosted user scope explicitly with `--user-scope`.

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `opt-in` | Enable autonomous submission after local redaction |
| `opt-out` | Disable autonomous submission |
| `status` | Show the local contribution policy |
| `preview` | Build a redacted contribution envelope from a recorded trace |
| `enqueue` | Queue a redacted envelope for autonomous submission |
| `flush-queue` | Submit eligible queued envelopes |
| `queue-status` | Show local autonomous queue diagnostics |
| `credit` | Show local credit totals and explanations |
| `submit` | Submit one redacted envelope to an explicit endpoint |
| `list-submissions` | List local submission records |
| `revoke` | Mark a submission revoked locally and optionally call a revocation endpoint |
| `ingest-health` | Check a Trace Commons ingestion service `/health` endpoint |
| `quarantine-list` | List centrally quarantined traces for reviewer/admin triage |
| `active-learning-review-queue` | List active-learning central trace review work |
| `review-decision` | Approve or reject a central quarantined trace |
| `review-lease-claim` | Claim a DB-backed central trace review lease |
| `review-lease-release` | Release a DB-backed central trace review lease |
| `append-credit-event` | Append a delayed central credit ledger event |
| `worker-utility-credit` | Append idempotent delayed utility credit through the worker route |
| `process-evaluation-submit` | Submit process-evaluation metadata and optional idempotent training utility credit |
| `analytics-summary` | Show central Trace Commons analytics |
| `maintenance-run` | Request central retention/revocation propagation and DB reconciliation maintenance |
| `worker-retention-maintenance` | Request scoped retention maintenance through the worker route |
| `retention-jobs-list` | List DB-backed retention maintenance jobs |
| `retention-job-items` | List DB-backed per-submission retention job actions |
| `worker-vector-index` | Request scoped vector indexing through the worker route |
| `benchmark-convert` | Request central benchmark conversion for approved replayable traces |
| `worker-benchmark-convert` | Request benchmark conversion through the worker route |
| `benchmark-lifecycle-update` | Update benchmark registry/evaluator lifecycle metadata |
| `replay-dataset-export` | Export approved replay dataset metadata |
| `replay-export-manifests` | List central replay export manifest metadata |
| `config-status` | Show safe non-secret Trace Commons config cutover/status booleans |
| `tenant-policy-get` | Read the DB-backed tenant contribution policy |
| `tenant-policy-set` | Write the DB-backed tenant contribution policy |
| `tenant-access-grants-list` | List DB-backed tenant access grants |
| `tenant-principal-ref` | Derive the stored principal_ref for tenant access grants |
| `tenant-access-grant-create` | Create a DB-backed tenant access grant |
| `tenant-access-grant-revoke` | Revoke a DB-backed tenant access grant |
| `ranker-training-candidates` | Export approved ranker training candidates |
| `ranker-training-pairs` | Export approved ranker training pairs |
| `audit-events` | List central Trace Commons audit events |
| `list-traces` | List central trace metadata with reviewer filters |
| `privacy-filter-canary` | Run a local Privacy Filter sidecar canary check |

**Local queue and credit behavior:**

- `flush-queue` submits eligible queued traces under the standing opt-in policy after compacting duplicate envelopes and orphan held sidecars.
- Retryable submission failures stay queued with typed, redacted backoff sidecars; due credit notices are emitted without exposing envelope bodies.
- Standing opt-in can use `IRONCLAW_TRACE_SUBMIT_TOKEN`, or guarded upload-claim issuer settings that mint short-lived EdDSA/Ed25519 claims for queue flush, explicit submit, status sync, and remote revoke.
- Upload-claim issuer URLs must be HTTPS, exactly host-allowlisted, redirect-free, and free of embedded credentials, query, or fragment values.
- `queue-status` reads local scoped state only and reports readiness, capture toggles, queue/held counts, retry and manual-review holds, compaction counters, sanitized warning aggregates, safe failure classes, and local credit summary.
- `credit --notice`, `--ack`, and `--snooze-hours` manage periodic local credit notices keyed by a safe credit fingerprint.
- Private ingestion status sync can reflect delayed ledger deltas and credit-event counts locally.

Central trace status filters accept `accepted`, `quarantined`, `rejected`, `revoked`, `expired`, and `purged`.

**Central reviewer/admin commands:**

- Central commands always require an explicit endpoint and `--bearer-token-env`; they do not change local opt-in defaults and do not upload raw traces.
- Health accepts a service base URL, `/health`, or `/v1/traces`; other central commands accept a service base URL or `/v1/traces` and derive private API paths.
- Missing credentials, non-2xx responses, and older servers that do not expose newer maintenance/review/export routes fail cleanly.
- Production upload claims should be EdDSA/Ed25519 signed; static tokens and HS256 claims remain controlled-pilot bridge paths.
- Ingestion services can use default/keyed EdDSA public keys, JSON/file/guarded-HTTPS keysets, activation windows, EdDSA-required auth, managed-keyset-only enforcement, and live issuer-managed keyset refresh.

**Review, maintenance, and export helpers:**

- `active-learning-review-queue` calls `/v1/review/active-learning` and exits normally with a not-exposed-yet message when older services return 404 or 501.
- `quarantine-list` and `active-learning-review-queue` accept `--lease-filter all|mine|available|active|expired`; lease claim/release commands manage `/v1/review/{submission_id}/lease`.
- `maintenance-run` posts to `/v1/admin/maintenance` with explicit `--purpose`, optional `--dry-run`, DB backfill/reconciliation/vector flags, retention purge options, and export-cache pruning controls.
- Worker routes provide narrower retention, vector-index, benchmark-convert, utility-credit, and process-evaluation requests.
- Benchmark, replay, ranker, analytics, config-status, audit, tenant-policy, and tenant-access commands expose compact metadata and safe readouts without raw trace bodies or secret material.
- Delayed credit commands require explicit reasons, and downstream utility events require `--external-ref` for auditability.
- `privacy-filter-canary` is local only; it requires `IRONCLAW_TRACE_PRIVACY_FILTER_COMMAND` and honors the documented timeout and IO-limit environment variables.

### completion

Generate shell completion scripts.

```bash
ironclaw completion <SHELL>
```

**Shells:** `bash`, `zsh`, `fish`, `powershell`

**Example:**

```bash
# Bash
ironclaw completion bash > /etc/bash_completion.d/ironclaw

# Zsh
ironclaw completion zsh > /usr/local/share/zsh/site-functions/_ironclaw
```

## Environment Variables

CLI behavior can be modified with environment variables:

```bash
# Config directory
export IRONCLAW_BASE_DIR=/custom/path

# Skip onboarding
export ONBOARD_COMPLETED=true

# Logging
export RUST_LOG=ironclaw=debug
```

| Code | Meaning |
|------|---------|
| `0` | Success |
| `1` | General error |
| `2` | Invalid arguments |
| `3` | Configuration error |
| `4` | Database error |
| `5` | Network error |
