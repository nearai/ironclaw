Install T3Claw directly on your local machine for personal use. This is the simplest and most common installation method.

## Installation by OS

<Tabs>
  <Tab title="Linux">
    ### Quick Install (Shell Script)

    ```bash
    curl -fsSL https://install.t3claw.ai | bash
    ```

    This installs to `~/.local/bin/t3claw`. Add to your PATH if needed:

    ```bash
    export PATH="$HOME/.local/bin:$PATH"
    echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
    ```

    ### Package Manager (Ubuntu/Debian)

    ```bash
    # Add repository
    curl -fsSL https://repo.t3claw.ai/gpg | sudo gpg --dearmor -o /usr/share/keyrings/t3claw.gpg
    echo "deb [signed-by=/usr/share/keyrings/t3claw.gpg] https://repo.t3claw.ai stable main" | sudo tee /etc/apt/sources.list.d/t3claw.list

    # Install
    sudo apt update
    sudo apt install t3claw
    ```

    ### Build from Source

    ```bash
    # Prerequisites
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    

    # Build
    git clone https://github.com/Terminal-3/t3-claw.git
    cd t3claw
    cargo build --release

    # Install
    sudo cp target/release/t3claw /usr/local/bin/
    ```

    ### Post-Install

    1. **Run the wizard**:
       ```bash
       t3claw onboard
       ```

    2. **Select libSQL** (recommended) during Step 1

    3. **Service setup** (optional):
       ```bash
       t3claw service install
       sudo systemctl enable --now t3claw
       ```
  </Tab>

  <Tab title="macOS">
    ### Homebrew (Recommended)

    ```bash
    # Add tap
    brew tap t3claw-ai/tap

    # Install
    brew install t3claw
    ```

    ### Shell Script

    ```bash
    curl -fsSL https://install.t3claw.ai | bash
    ```

    Installs to `~/.local/bin/t3claw`.

    ### Post-Install

    1. **Run the wizard**:
       ```bash
       t3claw onboard
       ```

    2. **macOS Keychain** (Step 2): Two system dialogs are expected:
       - "Enter your password to unlock the keychain"
       - "Allow t3claw to access this keychain item"

       Click "Always Allow" to minimize future prompts.

    3. **Service setup** (optional):
       ```bash
       t3claw service install
       brew services start t3claw
       ```
  </Tab>

  <Tab title="Windows (WSL2)">
    ### Prerequisites

    1. Install WSL2:
       ```powershell
       wsl --install
       ```

    2. Restart, then open Ubuntu in WSL2

    ### Install in WSL2

    ```bash
    # Inside WSL2
    curl -fsSL https://install.t3claw.ai | bash
    ```

    ### Access from Windows

    The Web Gateway binds to localhost by default:

    - In WSL2: `http://127.0.0.1:3000`
    - From Windows: `http://localhost:3000` (WSL2 forwards ports automatically)

    ### Post-Install

    ```bash
    t3claw onboard
    t3claw run
    ```

    <Note>
    WSL2 is the recommended way to run T3Claw on Windows. Native Windows support is available but less tested.
    </Note>
  </Tab>

  <Tab title="Windows (Native)">
    ### PowerShell Install

    ```powershell
    irm https://install.t3claw.ai | iex
    ```

    Or manually:
    1. Download `t3claw-x86_64-pc-windows-msvc.zip` from [releases](https://github.com/Terminal-3/t3-claw/releases)
    2. Extract to `C:\Program Files\T3Claw\`
    3. Add to PATH

    ### Manual PATH Setup

    ```powershell
    # Add to system PATH
    [Environment]::SetEnvironmentVariable(
      "Path",
      $env:Path + ";C:\Program Files\T3Claw",
      "User"
    )
    ```

    ### Post-Install

    ```powershell
    t3claw onboard
    t3claw run
    ```

    <Warning>
    Native Windows support is experimental. For production use, prefer WSL2 or Linux.
    </Warning>
  </Tab>
</Tabs>

## Database Recommendation

For local installs, use **libSQL** (embedded SQLite):

```
Wizard Step 1: Database Connection
→ Select "libSQL"
→ Path: ~/.t3claw/t3claw.db (default)
```

Benefits of libSQL for local use:
- No separate database server to install
- Zero configuration
- Automatic migrations
- Optional Turso cloud sync

PostgreSQL is available but adds complexity for single-user deployments.

## Keychain Behavior

Your OS manages the encryption master key:

<AccordionGroup>
  <Accordion title="macOS" icon="apple">
    Uses macOS Keychain. Two dialogs expected:
    1. "Enter your password to unlock the keychain"
    2. "Allow t3claw to access this keychain item"

    Click "Always Allow" on the second dialog to prevent repeated prompts.
  </Accordion>

  <Accordion title="Linux" icon="linux">
    Requires `gnome-keyring` or `kwallet`:

    ```bash
    # Ubuntu/Debian
    sudo apt install gnome-keyring

    # Fedora
    sudo dnf install gnome-keyring
    ```

    If unavailable, use environment variable mode in Step 2.
  </Accordion>

  <Accordion title="Windows" icon="windows">
    Uses Windows Data Protection API (DPAPI). No additional setup required.
  </Accordion>
</AccordionGroup>

## Service Installation

Run T3Claw as a background service:

<Tabs>
  <Tab title="Linux (systemd)">
    ```bash
    # Install service
    t3claw service install

    # Enable and start
    sudo systemctl enable --now t3claw

    # Check status
    sudo systemctl status t3claw

    # View logs
    sudo journalctl -u t3claw -f
    ```
  </Tab>

  <Tab title="macOS (launchd)">
    ```bash
    # Install service
    t3claw service install

    # Start with Homebrew
    brew services start t3claw

    # Or manually
    launchctl load ~/Library/LaunchAgents/ai.t3claw.service.plist

    # Check status
    launchctl list | grep t3claw
    ```
  </Tab>

  <Tab title="Windows">
    Windows service support is coming. For now, use:
    - Task Scheduler
    - NSSM (Non-Sucking Service Manager)
    - Or run in WSL2 with systemd
  </Tab>
</Tabs>

## Verify Installation

```bash
# Check version
t3claw --version

# Run diagnostics
t3claw doctor

# Expected output:
# ✓ Binary: t3claw v0.13.0
# ✓ Config directory: /home/user/.t3claw
# ✓ Database: libSQL (not yet initialized)
# ✓ Docker: available (optional)
```
