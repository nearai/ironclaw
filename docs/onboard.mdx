`ironclaw onboard` initializes the canonical Reborn runtime. It is a
non-interactive bootstrap command, not the retained v1 all-in-one setup wizard.

<Note>
If you have not installed IronClaw yet, start with the [Quickstart guide](/quickstart).
</Note>

## What the command does

```bash
ironclaw onboard
```

The command:

1. resolves `IRONCLAW_REBORN_HOME`, or defaults to `~/.ironclaw/reborn`;
2. creates the Reborn home directory;
3. creates missing `config.toml` and `providers.json` files;
4. preserves existing operator-edited files unless `--force` is passed;
5. provisions or preserves a mode-protected `webui-token` bearer-token file;
6. writes `.onboard-completed.json` with completed and pending setup steps.

It does not read or modify retained v1 state, configure a v1 database, or run
the v1 channel, tunnel, extension, sandbox, and heartbeat wizard.

## Finish provider setup

Choose a provider and model, then inspect the effective route:

```bash
ironclaw models list
ironclaw models set-provider openai --model gpt-5-mini
ironclaw models status
```

Set the API-key environment variable reported by `models list --verbose` before
starting the runtime. LLM/API credentials remain in environment variables and
are not written into `config.toml`; the generated WebUI bearer token is stored
in the mode-protected `webui-token` file.

## Inspect generated configuration

```bash
ironclaw config path
ironclaw config list
ironclaw config get boot.profile
```

Use `ironclaw config init --force` only when you intentionally want to replace
the generated configuration stubs.

## Start the runtime

```bash
ironclaw repl
ironclaw run --message "Hello"
```

For profiles, production storage, WebUI serve mode, and all command options,
see the [canonical CLI reference](/reborn-binary). The detailed onboarding
contract and current non-goals are documented in
[Reborn onboarding](/reborn/onboarding).

<Warning>
The retained v1 wizard is available only as `ironclaw-v1 onboard`. It operates
on v1 state and is not part of the canonical Reborn onboarding flow.
</Warning>
