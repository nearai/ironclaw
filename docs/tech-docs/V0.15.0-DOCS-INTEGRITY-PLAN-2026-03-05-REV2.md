# IronClaw v0.15.0 Docs Integrity Plan + Execution Report (2026-03-05, Rev2)

## Goal

Perform a release-scoped, source-grounded, and externally fact-checked documentation integrity review for `ironclaw-docs`, then publish a corrected docs release.

## Release Scope (strict)

- Source of truth: `~/src/ironclaw` at release tag `v0.15.0` (`3615967`, released 2026-03-04).
- Docs target: `~/src/ironclaw-docs` on `main`.
- Excluded: unreleased `main` commits in `ironclaw` after `v0.15.0`.

## Parallel Review Architecture

Four parallel agent tracks were run:

1. Release-consistency track: release tags/changelog vs docs metadata and summaries.
2. Core reference track: `AGENT_README.md` behavior and command/env references.
3. Integration track: `SIGNAL_SETUP.md`, `TELEGRAM_SETUP.md`, `BUILDING_CHANNELS.md`.
4. Developer/provider track: `DEVELOPER-REFERENCE.md`, `LLM_PROVIDERS.md`.

## Detailed Plan

1. Sync repos and tags (`fetch --all --tags --prune`).
2. Lock review baseline to `ironclaw@v0.15.0` and capture evidence refs.
3. Run parallel mismatch discovery on docs partitions.
4. Reconcile findings into a single correction matrix.
5. Apply minimal factual edits only; no speculative updates.
6. Re-validate corrected docs with targeted grep sweeps.
7. Cross-check externally published docs/pages for integration claims.
8. Commit, push, and publish new docs release tag.
